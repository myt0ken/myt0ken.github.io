构建一个MSF渗透VPS

VPS是virtual private server的简称。一般是具有一个公网IP的虚拟机。在渗透中具有公网IP的机器常常能派上用场，比如反弹一个shell。由于Windows的VPS价钱一般不太便宜，省点银子，一般用个linux的凑合凑合。

很多时候为了省更多一点银子，不会选用配置太高的VPS，一般以 1 vCPU && 1024 MiB的配置为主，这样有时会出现内存不足的情况，比如在安装一些比较大的软件和运行一些程序时，经常会出现 allocate memory failed的提示，这是运行内存不足造成的。为了解决这个问题，需要启用swap分区。

原理很简单，就是在硬盘上划分一块空间加入到运行内存中。RAM的I/O速度比硬盘这种外部存储要高很多，所以在使用虚拟内存运行程序时会感到运行速度的减慢。在硬盘上划分出的这块空间，在Windows上被称为一般被称为虚拟内存，在linux上被称为swap分区，但实际上是一个东西。

以ubuntu 18.04 为例

配置swap分区

1.查看swap分区是否建立

先用 free -m 或 free -h 查看内存状态，若显示swap为0，则表示没有swap分区。

使用 swapon 或 swapon -s 也可以查看swap分区配置情况。

2.创建swap分区文件

创建swap分区有很多方法，本人比较喜欢使用dd命令。

dd if=/dev/zero of=/swapfile count=2048 bs=1M #这是划分了2GiB的空间

选择放置swap分区文件的位置，一般是根目录下带有swap标识的文件。

ls / | grep swapfile，确认文件是否建立在根位置。

3.激活swap分区文件

swap不会自动激活，需要手动配置格式化和使能。需要用有R/W权限的root用户。

chmod 600 /swapfile，权限配置

ls -lh /swapfile，查看权限配置是否成功

mkswap /swapfile，格式化swap分区，会有回显

4.开启swap分区

swapon /swapfile，启用格式化过的分区文件

使用free -h查看swap记录，不为0就是配置好swap分区了

4.开机时自动加载swap分区文件

echo "/swapfile swap swap sw 0 0" >> /etc/fstab

这样swap分区就算是配置好了，也开启了开机自启动。

手工配置java环境

apt-get install openjdk可能是最简单的，但是open jdk对很多工具的兼容性有点蛋疼。长痛不如短痛，一般使用oracle jdk。

为什么下载一个oracle jdk包这么麻烦？在web网页上下载oracle jdk需要点击一个同意协议的按键才能下载，这是通过验证cookie做到的。 在命令行中就要费点事，不然只用wget下载回来的包是无法解压的。也不建议去非官方处下载，万一他给你加点料呢。

tar解压后

mv jdk1.8.0_162/ /usr/local/

ln -s /usr/local/jdk1.8.0_162/ /usr/local/jdk #设置软连接是方便修改环境变量

vim /etc/profile #编辑环境变量

source /etc/profile

java -version
